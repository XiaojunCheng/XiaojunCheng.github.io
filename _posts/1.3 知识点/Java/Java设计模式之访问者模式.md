# Java设计模式之访问者模式

## 一、基本概念

访问者模式是一种将算法与对象结构分离的软件设计模式

## 二、优缺点

- 优点
    1. 如果算法有变更，只需要改变访问者实现
    2. 添加新的对象结构，只需修改访问者接口以及其实现
- 缺点
    1. visit()方法的返回值的类型在设计系统式就需要明确，不然就需要修改访问者的接口以及所有接口实现。另外如果访问者接口的实现太多，系统的扩展性就会下降
